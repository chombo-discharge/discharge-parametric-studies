Writing Jobscripts

configurator.py schedules slurm jobs through sbatch expecting the bash script generic_array_job.sh. This should edited to suit the system configuration, and added as a job_script_dependency to the studies:

generic_array_job.sh
#!/bin/bash
# Author André Kapelrud
# Copyright © 2025 SINTEF Energi AS

#SBATCH --account=<cluster account>
#SBATCH --output=R-%x.%A-%a.out
#SBATCH --error=R-%x.%A-%a.err

# typical options needed for running on cluster
# remove leading '#' to use
##SBATCH --nodes=4 --ntasks-per-node=128
##SBATCH --partition=normal
##SBATCH --time=0-00:10:00

# Local slurm testing,
# add extra '#' to comment out
#SBATCH --ntasks=5 --cpus-per-task=1
#SBATCH --time=0-00:25:00

set -o errexit
set -o nounset

# example sigma2, module loading code
if command -v module > /dev/null 2>&1
then
    module restore system
    module load foss/2023a
    module load HDF5/1.14.0-gompi-2023a  # needed by chombo-discharge
    module load Python/3.11.3-GCCcore-12.3.0  # needed by jobscripts
fi

python ./jobscript_symlink  # run python through job-script
exit $?

The script configures the resource requirements, sets error conditions and loads sigma2 system modules (c.f. Lmod).

Note

It is possible to submit python scripts to sbatch directly if the python script has the correct shebang (#! /usr/bin/env python), the #SBATCH-specific comment directives also works from a python script. Routing through an intermediate bash-script made it somewhat easier to configure module-loading on sigma2.

A simple way of having two different versions of this bash script is to just make generic_array_job.sh a symlink to a correct system-tailored version, or even copy it in similar to the way chombo-discharge deals with the library makefiles.

Template jobscripts

At this stage the work is not done, because alot of of the heavy lifting has to be done by your jobscripts. Regard the configurator.py script as setting up the infrastructure. It is now up to you to start meaningful simulations. This section gives some examples on how to accomplish this.

Generic python jobscript example

A vanilla, quite simple python-based jobscript might look like this:

Database-dependent jobscript examples

This section contains two example jobscripts.

  • One for a database where the simulation (chombo discharge) is rerun under some condition.

  • One for a study that needs to extract some dataset from a database and set up sub directories per parameter space run.

The jobscripts depend on two python scripts: parse_report.py and config_util.py that needs to be included as job_script_dependencies in the run_definition.

Warning

These are not ready-to-run, but illustrates a concept. For specific examples see the actual source code listings of this project.

 48    dname = [f for f in os.listdir() if (os.path.isdir(f) and re.match(dpattern, f))][0]
 49

 51    log.info(f'chdir: {dname}')

 55    input_file = None

 66    # invocation that is not taken care of in your *.inputs file, you can add

 81    # above.
 82

 85    def parse_some_result_file(result_filename):
 86        """ meaningless stub """
 87        return None
 88
 89    data = parse_some_result_file(result_fn)
 90    log.info("Some description of this step...")
 91
 92    def calculate_interresting_value(data):
 93        """ meaningless stub """
 94        return None  # stub
 95    iv = calculate_interresting_value(data)
 96
 97    # If we need to read something from a *.inputs file we can of course do that:
 98    orig_iv = read_input_float_field(input_file, 'SomeNamespace.interrestingvariable')
 99    if orig_iv None:
100        raise RuntimeError(f"'{input_file}' does not contain 'SomeNamespace.interrestingvariable' field")
101
102    # some decision point
103    if orig_iv != iv:
104        log.info(f'renaming: {result_fn} -> {result_fn}.0')
105        shutil.move(result_fn, f'{result_fn}.0')
106
107        # You might want to back up other results here.
108
109        # update input file, this mirrors the run_definition file syntax
110        handle_combination({
111            "interresting_parameter": {  # parameter name
112                "target": input_file,  # file target
113                "uri": "SomeNamespace.interrestingvariable"
114                }
115            }, dict(interresting_parameter=iv))
116
117        # rerun the simulation!
118        log.info('Rerunning calculations')
119        p = subprocess.Popen(cmd, shell=True, executable="/bin/bash")
120        while p.poll() is None:
121            time.sleep(0.5)

Study (database-dependent) jobscript example

This is a rather long example where we traverse the database directories to find relevant data and then set up detailed simulations to use that data. This can be a single simulation, or a complex-subhierarchy of simulations. The last part is only pseudo-code, so the reader is advised to check out some of the checked in example studies in the main repository.

  1#!/usr/bin/env python
  2"""
  3Author André Kapelrud
  4Copyright © 2025 SINTEF Energi AS
  5"""
  6
  7import os
  8import sys
  9import json
 10import re
 11import logging
 12import itertools
 13import shutil
 14
 15from subprocess import Popen, PIPE
 16
 17from pathlib import Path
 18
 19# local imports
 20sys.path.append(os.getcwd())  # needed for local imports from slurm scripts
 21from parse_report import parse_report_file  # noqa: E402
 22from config_util import (  # noqa: E402
 23                         copy_files, backup_file,
 24                         get_slurm_array_task_id,
 25                         handle_combination,
 26                         DEFAULT_OUTPUT_DIR_PREFIX
 27                         )
 28
 29
 30if __name__ == '__main__':
 31
 32    log = logging.getLogger(sys.argv[0])
 33    formatter = logging.Formatter(
 34            '%(asctime)s | %(levelname)s :: %(message)s')
 35    sh = logging.StreamHandler(sys.stdout)
 36    sh.setFormatter(formatter)
 37    log.addHandler(sh)
 38    log.setLevel(logging.INFO)
 39
 40    task_id = get_slurm_array_task_id()
 41    log.info(f'found task id: {task_id}')
 42
 43    with open('structure.json') as structure_file:
 44        structure = json.load(structure_file)
 45
 46    # extract the directory prefix of run directories (default is 'run_',
 47    # but make no assumptions.
 48    job_prefix = 'run_'
 49    if 'output_dir_prefix' in structure:
 50        job_prefix = structure['output_dir_prefix']
 51
 52    dpattern = f'^({job_prefix}[0]*{task_id:d})$'  # account for possible leading zeros
 53    dname = [f for f in os.listdir() if (os.path.isdir(f) and re.match(dpattern, f))][0]
 54    log.info(f'chdir: {dname}')
 55    os.chdir(dname)
 56
 57    # locate .inputs file (should be in the required_files list, and copied to the
 58    # current directory):
 59    input_file = None
 60    for f in os.listdir():
 61        if os.path.isfile(f) and f.endswith('.inputs'):
 62            input_file = f
 63            break
 64    if not input_file:
 65        raise ValueError('missing *.inputs file in run directory')
 66    log.info(f"input file: {input_file}")
 67
 68    # get access to structure of dependent database through symlink
 69    with open('../inception_stepper/structure.json') as db_structure_file:
 70        db_structure = json.load(db_structure_file)
 71
 72    # determine order of parameters in database (might differ from the order in this study)
 73    if 'space_order' not in db_structure:
 74        raise ValueError("missing field 'space_order' in database 'inception_stepper'")
 75    db_param_order = db_structure['space_order']
 76
 77    # load this run's parameters (radius, pressure, etc.)
 78    with open('parameters.json') as param_file:
 79        parameters = json.load(param_file)
 80
 81    # we can add run-time checks:
 82    if 'geometry_radius' not in parameters:
 83        raise RuntimeError("'geometry_radius' is missing from 'parameters.json'")
 84
 85    # put the parameters in the same order as the database index needs them
 86    db_search_index = []
 87    for db_param in db_param_order:
 88        db_search_index.append(parameters[db_param])
 89
 90    # Now, we need to locate the corresponding data in the 'database':
 91    with open('../inception_stepper/index.json') as db_index_file:
 92        db_index = json.load(db_index_file)
 93
 94    # linear search through index, which is a dictionary.
 95    index = -1
 96    for db_i, params in db_index['index'].items():
 97        if params == db_search_index:
 98            index = int(db_i)
 99            break
100    if index < 0:
101        raise RuntimeError(f'Unable to find db parameter_set: {db_param_order} = ' +
102                           f'{db_search_index}')
103    log.info(f"Found database parameters {db_param_order} = {db_search_index} "
104             f"at index: {index}")
105
106    # we have the index, now locate the correct subdirectory:
107    db_run_path = Path('../inception_stepper')
108    if 'prefix' in db_index:
109        db_run_path /= db_index['prefix'] + str(index)
110    else:
111        db_run_path /= DEFAULT_OUTPUT_DIR_PREFIX + str(index)
112
113
114    def parse_and_get_interresting_data(filename):
115        """ stub """
116        return None
117
118    data = parse_and_get_interresting_data(db_run_path / '<some-database-result-file>')
119
120    # Maybe we need to do some selective picking of data based on a dummy-parameter?
121    # here we can for easily check a 'dummy' parameter
122    if 'dummy-parameter' in parameters:
123        data = some_filter_action(parameters['dummy-parameter']  #do something meaningful
124
125    #----------------------------------------------------------------------------
126    # At this point we can do whatever we like with the data.
127
128    # Maybe the database study gave an estimate of a parameter and we just
129    # want to write that parameter to an *.inputs file or *.json file and run
130    # a detailed simulation.
131    # If so, use the handle_combination() function to write the data and
132    # launch the job using mpirun. See database example.
133
134    # If on the other hand, the database produces e.g. a voltage list and some
135    # other associated input data, then we need to create a sub-file
136    # hierarchy in the run-directory and submit those simulation jobs to slurm.
137
138    # We utilize helper functions from the configurator to alleviate the burden.
139
140    # We will use the generic_array_job_jobscript.py script at the leaf directory level.
141    #----------------------------------------------------------------------------
142
143    # let us enumerate the interresting data, assuming some known structure:
144    #   data[i] corresponds to the (new) parameters ["voltage", "some-other-parameter"]
145    enum_table = list(enumerate(data))
146
147    output_prefix = "voltage_"
148
149    # first we have to create an index for the sub-directories:
150
151    # guard for reposting of the job
152    MAX_BACKUPS = 10
153    index_path = Path('index.json')
154    backup_file(index_path, max_backups=MAX_BACKUPS)
155
156    # write voltage index
157    with open(index_path, 'w') as voltage_index_file:
158        json.dump(dict(
159            key=["voltage", "some-other-parameter"],
160            prefix=output_prefix,
161            index={i: item for i, item in enum_table}
162            ),
163                  voltage_index_file, indent=4)
164
165    # recreate the generic job-script symlink, so that the actual .sh jobscript work:
166    if not os.path.islink('jobscript_symlink'):
167        os.symlink('generic_array_job_jobscript.py', 'jobscript_symlink')
168
169    # create run directories, copy files, set voltage and parameters, etc.
170    for i, row in enum_table:
171        voltage_dir = Path(f'{output_prefix}{i:d}')
172        # don't delete old invocations
173        backup_dir(voltage_dir, max_backups=MAX_BACKUPS)
174        os.makedirs(voltage_dir, exist_ok=False)
175
176        # further symlink program executable to this directory's program-symlink
177        link_path = voltage_dir / 'program'
178        if not link_path.is_symlink():
179            os.symlink(Path('../program'), link_path)
180
181        # grab original file names from structure
182        required_files = [Path(f).name for f in structure['required_files']]
183        copy_files(log, required_files, voltage_dir)
184
185        # reuse the combination writing code from the configurator / config_util, by
186        # building a fake combination and parameter space:
187        # populate values
188        comb_dict = dict(
189                voltage=row[0],
190                some_other_parameter=row[1]
191                )
192        pspace = {
193                "voltage": {
194                    "target": voltage_dir/input_file,
195                    "uri": "SomeNamespace.potential",
196                    },
197                "some_other_parameter": {
198                    "target": voltage_dir/'chemistry.json',
199                    'uri': [ ... ]  # some very complex JSON traversing uri
200                    },
201                }
202        handle_combination(pspace, comb_dict)
203
204    # all voltage_* directories are now ready, and we can post a (new!) slurm array job:
205    cmdstr = f'sbatch --array=0-{len(enum_table)-1} ' + \
206            f'--job-name="{structure["identifier"]}_voltage" ' + \
207            'generic_array_job.sh'
208    log.debug(f'cmd string: \'{cmdstr}\'')
209    p = Popen(cmdstr, shell=True, stdout=PIPE, encoding='utf-8')
210
211    job_id = -1
212    while True:  # wait until sbatch is complete
213        # try to capture the job id
214        line = p.stdout.readline()
215        if line:
216            m = re.match('^Submitted batch job (?P<job_id>[0-9]+)', line)
217            if m:
218                job_id = m.groupdict()['job_id']
219
220                array_job_id_path = Path('array_job_id')
221                # backups for previously posted runs:
222                backup_file(array_job_id_path, max_backups=MAX_BACKUPS)
223
224                # write array index file
225                with open(array_job_id_path, 'w') as job_id_file:
226                    job_id_file.write(job_id)
227                log.info(f"Submitted array job (for '{structure['identifier']}" +
228                         f"_voltage' combination set). [slurm job id = {job_id}]")
229
230        if p.poll() is not None:
231            break
